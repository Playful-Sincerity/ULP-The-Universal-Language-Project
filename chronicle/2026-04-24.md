**Means:** `gh repo rename` for GitHub-side renames + `git remote set-url` to update local tracking; `git rm -r --cached` for the directory removals; targeted Edits across the five public docs for the cross-reference updates.

**Bug surfaced + fixed:** The first removal commit did not actually remove the files. Cause: a subsequent `git add -A` in the same staging cycle re-indexed the files that `git rm --cached` had just removed from the index. The doc updates landed but the deletions were quietly reverted. Caught the inconsistency on the next push verification (commit count vs expected file count mismatch), then did a clean follow-up commit `a3a1bbb` with `git rm --cached` followed immediately by `git commit` (no `git add` in between). End state: 0 files tracked under `2d-writer/` or `knowledge/sources/wisdom-speech/`, files preserved on disk locally.

**Lesson:** Do not run `git add -A` after `git rm --cached` in the same staging cycle — the latter undoes the former. Either commit between them, or use `git rm` (without `--cached`) so the files leave the working tree as well.

**`.gitignore` decision.** Wisdom asked whether `.gitignore` belongs in a public repo at all. Confirmed yes — it is the file that tells git what NOT to track, conventionally committed in every public repo. Contains no secrets; just patterns. Decision: keep the existing `.gitignore` as-is. It is doing real safety work — the `connections/artifact.md` entry blocks accidental commit of Dennis Hansen's private collaboration notes via any future `git add -A` — at no public cost.
